Dependency Mapping - Service graph của maplibre-native

Tài liệu này mô tả graph dịch vụ/runtime subsystem của maplibre-native dựa trên source hiện tại trong maplibre-native/. “Service” ở đây là các khối thực thi lớn trong SDK native, không phải microservice mạng.

1. Bức tranh tổng quan

Luồng chính bắt đầu từ SDK platform tạo mbgl::Map. Map::Impl sở hữu style::Style, Transform, AnnotationManager, giữ FileSource dùng chung và đăng ký chính nó làm style::Observer + RendererObserver. Mỗi lần trạng thái thay đổi, Map::Impl::onUpdate() đóng gói UpdateParameters gồm snapshot style immutable, camera, sprite/glyph state, nguồn file và gửi sang RendererFrontend.

# Dependency Mapping - DB dependency của `maplibre-native`

Tài liệu này mô tả dependency dữ liệu SQLite/offline/cache trong `maplibre-native`, tập trung vào `DatabaseFileSource`, `OfflineDatabase`, `OfflineDownload`, schema và lifecycle cache/offline.

## 1. Bản đồ DB runtime

```mermaid
flowchart TD
  Requester["Style / Tile / Glyph / Sprite requester"] --> MRL["MainResourceLoader"]
  MRL -->|cache-capable request| DFS["DatabaseFileSource"]
  DFS --> DBThread["DatabaseFileSourceThread"]
  DBThread --> ODB["OfflineDatabase"]
  ODB --> SQLite["mapbox::sqlite::Database"]
  SQLite --> Tables["resources, tiles, regions,<br/>region_resources, region_tiles"]
```

`DatabaseFileSource` là adapter async public cho DB. `OfflineDatabase` là implementation SQLite thực tế. `MainResourceLoader` dùng `DatabaseFileSource` cho ambient cache và dùng `forward()` để ghi response network vào DB. Offline region API cũng đi qua `DatabaseFileSource`.

### 3.1 `resources`

Chứa vector/raster/DEM tiles. Khóa unique là `(url_template, pixel_ratio, z, x, y)`, không phải URL đã substitute.

| Cột | Ý nghĩa |
|---|---|

# Dependency Mapping - Protocol dependency của `maplibre-native`

Tài liệu này mô tả dependency protocol/URL/resource routing trong `maplibre-native`: `asset://`, `file://`, `mbtiles://`, `pmtiles://`, HTTP/HTTPS, canonical tile server URLs, resource kinds, cache/network loading methods, Android HTTP bridge và cURL default path.

## 1. Protocol routing tổng quát

```mermaid
flowchart TD
  Resource["Resource(kind,url,loadingMethod)"] --> MRL["MainResourceLoader"]
  MRL --> Asset{"asset:// ?"}
  Asset -->|yes| AssetFS["AssetFileSource"]
  Asset -->|no| MBTiles{"mbtiles:// ?"}
  MBTiles -->|yes| MBTilesFS["MBTilesFileSource"]
  MBTiles -->|no| PMTiles{"pmtiles:// ?"}
  PMTiles -->|yes| PMTilesFS["PMTilesFileSource"]
  PMTiles -->|no| File{"file:// ?"}
  File -->|yes| LocalFS["LocalFileSource"]
  File -->|no| Cache{"Cache allowed?"}
  Cache -->|yes| DBFS["DatabaseFileSource"]
  DBFS -->|miss/stale| Network{"Network allowed?"}
  Cache -->|no| Network
  Network -->|yes| OnlineFS["OnlineFileSource"]
  OnlineFS --> HTTPFS["HTTPFileSource"]
  Network -->|no| Error["Unsupported/noContent"]
```

Thứ tự waterfall trong `MainResourceLoaderThread::request` là: asset, MBTiles, PMTiles, local file, database cache, network. Database cache chỉ request nếu `Resource::LoadingMethod` có `Cache`; network chỉ request nếu có `Network` và `OnlineFileSource::canRequest`.

## 2. Protocol constants và owner

| Protocol / scheme | Constant hoặc nhận diện | Owner | Ý nghĩa |
|---|---|---|---|
| `asset://` | `util::ASSET_PROTOCOL` | `AssetFileSource`, Android `AssetManagerFileSource` | File đóng gói trong app/assets. |
| `file://` | `util::FILE_PROTOCOL` | `LocalFileSource` | File path local sau percent decode. |
| `mbtiles://` | `util::MBTILES_PROTOCOL` | `MBTilesFileSource` | SQLite MBTiles file, generate TileJSON hoặc đọc tile. |
| `pmtiles://` | `util::PMTILES_PROTOCOL` | `PMTilesFileSource` | PMTiles v3 archive, hỗ trợ byte range qua nested ResourceLoader. |
| `http://`, `https://` | URL thường | `OnlineFileSource` + `HTTPFileSource` | Network resources. |
| `maplibre://` | `TileServerOptions::uriSchemeAlias()` default MapLibre | `util::mapbox::*normalize*URL` | Canonical style/source/tile/sprite/glyph URL chuyển sang base URL. |
| `mapbox://` | Mapbox configuration | `TileServerOptions::MapboxConfiguration` | Canonical Mapbox URL, có access token query. |
| `local://` Android | Java `NativeHttpRequest` special case | Android render tests | Load từ Android assets qua `LocalRequestTask`. |

## 4. Tile URL token dependency

`Resource::tile` thay token từ URL template:

| Token | Giá trị |
|---|---|
| `{z}` | Zoom level. |
| `{x}` | Tile x. |
| `{y}` | Tile y, đã đảo nếu source scheme là TMS. |
| `{quadkey}` | Bing quadkey tính từ x/y/z. |
| `{bbox-epsg-3857}` | Bounding box Web Mercator của tile. |
| `{prefix}` | Hai ký tự hex từ x/y để shard path. |
| `{ratio}` | `@2x` nếu pixel ratio > 1 và template hỗ trợ. |

Cache key tile trong DB không chỉ là URL đã thay token; DB lưu `url_template`, `pixel_ratio`, `z`, `x`, `y`. Điều này làm cache ổn định với cùng template và tile coordinate.

## 5. Loading methods và storage policy

| Field | Values | Ảnh hưởng routing |
|---|---|---|
| `Resource::LoadingMethod` | `CacheOnly`, `NetworkOnly`, `All` | Quyết định `DatabaseFileSource` và `OnlineFileSource` có được gọi không. |
| `Resource::Usage` | `Online`, `Offline` | Android HTTP nhận `offlineUsage`; online file source ưu tiên/throttle offline có thể khác. |
| `Resource::Priority` | `Regular`, `Low` | `OnlineFileSource` queue ưu tiên regular trước low. Stale cache usable thường revalidate low priority. |
| `Resource::StoragePolicy` | `Permanent`, `Volatile` | `Volatile` không đọc/ghi DB cache. |
| `Resource::dataRange` | byte range pair | HTTP Range header; PMTiles dùng rất nhiều. |
| `priorEtag/priorModified/priorExpires/priorData` | cache metadata | Conditional HTTP request và 304 handling. |

## 6. HTTP/HTTPS path

```mermaid
sequenceDiagram
  participant MRL as MainResourceLoader
  participant Online as OnlineFileSource
  participant HTTP as HTTPFileSource
  participant Server

  MRL->>Online: request(Resource)
  Online->>Online: resourceTransform? queue? max concurrent?
  Online->>HTTP: request(transformed Resource)
  HTTP->>Server: GET + Range/If-None-Match/If-Modified-Since
  Server-->>HTTP: status + headers + body
  HTTP-->>Online: Response
  Online-->>MRL: Response or retry/backoff
  MRL->>MRL: forward to DB if cacheable
```

Default platform dùng cURL multi + run loop socket watches trong `platform/default/src/mbgl/storage/http_file_source.cpp`. Android dùng JNI `HTTPFileSource` tạo `org.maplibre.android.http.NativeHttpRequest`, rồi Java module `HttpRequest` thực hiện transport.

### 6.1 HTTP status mapping

| Status / lỗi | `Response` |
|---|---|
| `200`, `206` | `data` có body. |
| `204` | `noContent = true`. |
| `304` | `notModified = true`. |
| `404` tile | `noContent = true` để tương thích. |
| `404` non-tile | `Response::Error::NotFound`. |
| `429` | `Response::Error::RateLimit`, parse `Retry-After` / `X-Rate-Limit-Reset`. |
| `5xx` | `Response::Error::Server`. |
| connection failure | `Response::Error::Connection`. |
| khác | `Response::Error::Other`. |

### 6.2 Cache-control fields

| Header / field | Mapping |
|---|---|
| `ETag` | `Response::etag`, sau đó DB `etag`. |
| `Last-Modified` | `Response::modified`, sau đó DB `modified`. |
| `Cache-Control` | `Response::expires`, `mustRevalidate`. |
| `Expires` | Override/đặt `Response::expires`. |
| `Retry-After`, `X-Rate-Limit-Reset` | `Response::Error::retryAfter`. |
| `Range` | Request header từ `Resource::dataRange`. |

`src/mbgl/util/mapbox.cpp` xử lý normalize/canonicalize URL dựa trên `TileServerOptions`.

| Function | Input canonical | Output normalized | Dùng cho |
|---|---|---|---|
| `normalizeStyleURL` | `maplibre://maps/style`, `mapbox://styles/...` | Base URL + style template + API key nếu cần | Style load |
| `normalizeSourceURL` | canonical source URL | TileJSON/source URL thực tế | Source load |
| `normalizeTileURL` | canonical tile URL | Tile URL thực tế | Tile request |
| `normalizeSpriteURL` | canonical sprite URL | Sprite URL thực tế | Sprite JSON/image |
| `normalizeGlyphsURL` | canonical glyph URL | Glyph URL thực tế | Glyph PBF |
| `canonicalizeTileURL` | normalized HTTP tile URL | canonical URL trừ token | Cache/source canonicalization |

MapLibre default style trong `tile_server_options.cpp` dùng `maplibre://maps/style` và base URL `https://demotiles.maplibre.org`. Mapbox configuration dùng `https://api.mapbox.com` và `mapbox://...` canonical scheme.

## 8. Local protocols

### 8.1 `asset://`

`AssetFileSource` nhận URL bắt đầu `asset://`, bỏ prefix, percent-decode path và prefix bằng `ResourceOptions::assetPath()`. Android có source riêng dựa trên `AssetManager`.

Path shape:

```text
asset://styles/basic.json -> <assetPath>/styles/basic.json
```

### 8.2 `file://`

`LocalFileSource` nhận URL bắt đầu `file://`, bỏ prefix, percent-decode thành path local rồi đọc bằng `requestLocalFile`.

Path shape:

```text
file:///tmp/style.json -> /tmp/style.json
```

### 8.3 `mbtiles://`

`MBTilesFileSource` đọc SQLite MBTiles:

| Request | Hành vi |
|---|---|
| `Resource::Source` | Mở DB read-only, đọc `metadata`, tạo TileJSON. |
| `Resource::Tile` | Query `tiles.tile_data` với `zoom_level`, `tile_column`, `tile_row`; đảo y theo MBTiles/TMS; decompress nếu cần. |

TileJSON generated thêm `tiles: [resource.url + "?file={x}/{y}/{z}.<format>"]`, set `tilejson = 2.0.0`, `scheme = xyz`, `encoding = mlt` nếu format là MLT.

### 8.4 `pmtiles://`

`PMTilesFileSource` đọc PMTiles v3:

| Giai đoạn | Hành vi |
|---|---|
| Header | Request byte range `0-126`, deserialize `pmtiles::headerv3`. |
| Metadata | Dùng header offsets để request metadata range, deserialize JSON. |
| Directory | Duyệt root/leaf directory để tìm tile address theo tile id. |
| Tile | Request byte range tile data qua nested `ResourceLoader`, decompress gzip nếu header yêu cầu. |

PMTiles giữ cache trong memory: header cache, metadata cache, directory cache có giới hạn entry. URL bên trong `pmtiles://` có thể là HTTP hoặc `file://`, vì nested request đi qua `FileSourceType::ResourceLoader`.

## 9. Android HTTP bridge

```mermaid
flowchart LR
  Cpp["HTTPFileSource::request"] --> Native["NativeHttpRequest JNI peer"]
  Native --> Java["org.maplibre.android.http.NativeHttpRequest"]
  Java --> Module["MapLibre.getModuleProvider().createHttpRequest()"]
  Module --> Impl["HttpRequest implementation"]
  Impl --> Responder["HttpResponder.onResponse/handleFailure"]
  Responder --> CppCallback["nativeOnResponse/nativeOnFailure"]
  CppCallback --> Response["mbgl::Response"]
```

Android protocol notes:

| Detail | Ý nghĩa |
|---|---|
| `dataRange` string | Tạo từ `Resource::dataRange` dạng `bytes=start-end`. |
| `etag` hoặc `modified` | Conditional request header. |
| `offlineUsage` | Java implementation có thể thêm `offline=true` hoặc policy riêng. |
| `local://` | Shortcut render-test để đọc assets bằng `LocalRequestTask`. |
| native pointer lock | Java lock tránh callback vào native request đã cancel. |

## 10. Protocol risk hotspots

| Hotspot | Rủi ro |
|---|---|
| Waterfall order | `mbtiles://`/`pmtiles://`/`file://` phải được xử lý trước DB/network, nếu không URL local có thể bị route sai. |
| Cache-only request | Nếu source không hỗ trợ cache-only hoặc loadingMethod sai, tile placeholder/cache warmup bị ảnh hưởng. |
| PMTiles nested ResourceLoader | Có thể tạo vòng routing nếu URL inner vẫn là `pmtiles://`; implementation extract URL và request inner URL nên cần giữ invariant này. |
| HTTP 404 tile as noContent | Behavior đặc biệt này tránh coi tile missing là fatal; thay đổi có thể làm nhiều style fail. |
| Canonical URL + API key | Cache key phải tránh nhúng access token; normalization/canonicalization sai làm cache miss hoặc lộ token. |
| Android vs default HTTP | Cùng contract `Response`, nhưng timeout/cache/offline query parameter có thể khác implementation Java/cURL. |
| Range requests | PMTiles phụ thuộc server/local source hỗ trợ `dataRange`; lỗi range làm header/tile parse fail. |
